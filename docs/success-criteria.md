# Portal Success Criteria

## Renter

- understands the property offer quickly
- sees approved information and uncertainty clearly
- can inquire without confusion
- can self-serve common stay or rental information
- can report support or maintenance issues
- can identify urgent escalation language

## Owner

- sees listing drafts awaiting approval
- sees the automation boundary
- can review support and inquiry flow
- can use the ops page as a weekly rhythm
- does not need to edit code for every content change once wired to the owner workspace

## Template

- validates without secrets
- builds on Vercel
- runs with sample data
- makes manual v1 boundaries explicit
- can be installed for another owner without brother-specific data
